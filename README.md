# Library Management System SQL Project

## Overview
The Library Management System is a SQL-based database project developed using PostgreSQL and pgAdmin 4. It is designed to manage library operations such as maintaining book records, member information, and book issue/return transactions.

## Objectives
- Store and manage library data efficiently.
- Track books available in the library.
- Manage member records.
- Monitor book issue and return activities.
- Generate reports using SQL queries.

## Database Tables
### Books
Stores information about books available in the library.

### Members
Stores details of registered library members.

### Issued_Books
Maintains records of books issued to members.

## Features
- Add, update, and delete book records.
- Manage member information.
- Track issued and returned books.
- Search books by title, author, or category.
- Generate reports using SQL queries.
- Perform data analysis using aggregate functions.

## SQL Concepts Used
- SELECT Statements
- WHERE Clause
- ORDER BY
- GROUP BY
- HAVING
- Aggregate Functions (SUM, COUNT, AVG)
- JOINS
- Subqueries
- Constraints
- Primary Keys and Foreign Keys

## Sample Queries
- Retrieve all available books.
- Find books issued to a specific member.
- Count total books in the library.
- Display members who have borrowed books.
- Generate issue and return reports.

## How to Run
1. Open PostgreSQL/pgAdmin 4.
2. Create a new database.
3. Execute the `schema.sql` file to create tables.
4. Run the queries from `queries.sql`.
5. View and analyze the results.
